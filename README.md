# 1C Server Docker

Этот проект предназначен для сборки Docker-образа сервера 1С:Enterprise на базе Ubuntu с установленным серверным компонентом 1С.

## Что делает проект

Проект автоматизирует процесс:
- сборки базового контейнера с необходимыми системными зависимостями;
- установки сервера 1С из установочного файла `setup-full-*.run`;
- подготовки готового Docker-образа и архива образа для переноса на другой хост.

## Структура проекта

- `build_images.sh` — основной скрипт сборки. Извлекает версию из имени установочного файла, собирает образ и сохраняет его в архив.
- `Docker/onec_base/Dockerfile` — базовый образ с Ubuntu, русской локалью и зависимостями для 1С.
- `Docker/server/Dockerfile` — образ сервера 1С, который устанавливает серверные компоненты и подготавливает запуск.
- `Docker/server/entrypoint.sh` — точка входа, запускающая `ras` и `ragent`.

## Требования

- Docker
- Linux/WSL или другая среда с поддержкой Docker
- Установочный файл 1С в формате `setup-full-<версия>-x86_64.run`

## Как собрать

1. Поместите установочный файл в каталог:
   - `Docker/server/setup-full-8.3.27.2214-x86_64.run`
2. Запустите:

```bash
sh ./build_images.sh
```

Скрипт:
- определит версию из имени файла;
- соберёт образ `onec_base`;
- соберёт образ `1cserver:<version>`;
- сохранит архив `1cserver-<version>.tar`.

## Как запустить

После загрузки архива на целевой сервер выполните:

```bash
docker load -i /path/to/1cserver-<version>.tar
```

Пример запуска контейнера:

```bash
docker run -d \
  --name 1cserver-<version> \
  --restart unless-stopped \
  --network host \
  -v /home/usr1cv8/.1cv8:/home/usr1cv8/.1cv8 \
  -v /var/1C/licenses:/var/1C/licenses \
  -v /etc/localtime:/etc/localtime:ro \
  -v /etc/timezone:/etc/timezone:ro \
  -v /_SHARE/exchange:/_SHARE/exchange \
  1cserver:<version>
```

## Примечания

- В проекте используются файлы установщика 1С, поэтому их не стоит хранить в репозитории без необходимости.
- Для работы контейнера нужны доступные каталоги для файловой базы и лицензий.
